# GarageFlow App

GarageFlow App is an early-stage public portfolio project for a small full-stack garage workflow application.

The goal is to help independent garages replace paper job notes with a clean digital workflow for customers, vehicles, bookings, mechanic notes, parts, labour, costs, vehicle history and printable customer handover summaries.

This scaffold uses local The Software House boilerplates:

- `backend/` is based on the TSH Express boilerplate.
- `frontend/` is based on the TSH React starter boilerplate.

The project is intentionally small. It is not a garage ERP, invoicing system, payment platform or multi-tenant SaaS.

## Current Status

Scaffold only. GarageFlow domain features have not been implemented yet.

## Planned Stack

- Backend: Express, TypeScript, TypeORM, PostgreSQL, Celebrate/Joi, Mocha/Chai/Supertest
- Frontend: React, Vite, TypeScript, TanStack Router, TanStack Query, Axios, Vitest, Playwright

## Planned Core Features

- Customer records
- Vehicle records
- Registration number lookup
- Mechanics
- Jobs/bookings
- Job status workflow
- Mechanic notes
- Parts and labour items
- Automatic cost totals
- Vehicle/job history
- Printable customer handover summary
- Seed data
- Backend API tests
- Playwright E2E tests
